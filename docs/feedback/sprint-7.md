## テスト結果詳細

### 合格した項目

- **S7-1 夫婦優先度タブの表示・遷移**: ヘッダー下の「👫 夫婦優先度」トグルが表示され、クリックで夫婦優先度ページに遷移。ページ内に「夫」「妻」「比較」の3サブタブが表示され切替可能。
- **S7-2 夫タブの優先度入力**: 固定11項目（駅からの距離・物件価格・広さ間取り・新築かどうか・管理費修繕・日当たり・収納・駐車場・学校保育園・買い物・静かさ治安）が全て表示。★3クリックで優先度3、★5クリックで優先度5に設定（aria-pressed で検証）。同じ星を再クリックで0（未入力）に戻ることを確認。
- **S7-3 妻タブの独立入力**: 妻の駅★5を設定後、夫タブに戻ると夫の駅は★3のまま維持。夫・妻の値が混在しないことを aria-pressed で厳密検証（夫=3 / 妻=5）。
- **S7-4 カスタム項目の追加・削除**: 「バルコニーの広さ」を追加 → 夫・妻・比較の3タブすべてに表示（各 count=1）。削除ボタンで3タブすべてから消去（夫=0 妻=0 比較=0）。固定項目・他の値は保持。
- **S7-5 比較タブの差分強調**: 夫駅★2 / 妻駅★5（差3）の行が `bg-amber-50 border-l-2 border-amber-400` で強調され ⚠ アイコン表示。夫価格★3 / 妻価格★3（差0）の行は ✓（一致）アイコンを表示しアンバー強調なし。★文字列表示（夫駅=★★☆☆☆）も正常。
- **S7-6 Firestore 保存・リロード後の保持**: 設定した優先度がページリロード後も `shared/main` 共有ドキュメントから復元され保持（夫駅★2維持を確認）。
- **S7-7 ローン計算側との独立性（回帰）**: ローン計算ページで物件価格を変更しても夫婦優先度（夫駅★2）は不変。逆方向も `coupleConditions` が別系統データのため非干渉。
- **S7-8 オフライン時のブロック**: DevTools 相当のオフライン設定で「⚠ オフライン中 — 読み取り専用」バナー表示。夫婦優先度の★ボタンが disabled 化（`starDisabled=true`）。閲覧・サブタブ切替は継続可能。

## 補足（テスト上の留意点・バグではない）

- `shared/main` は全利用者共有の単一ドキュメントのため、テスト実行間で `coupleConditions` の状態が引き継がれる。初回の素朴なテストで★トグル仕様（既設定の星を再クリックすると0に戻る）と相互作用して見かけ上の不一致が出たが、状態を読み取って目標値に設定する決定論的テストで全項目 PASS を再確認済み。アプリの挙動は仕様通り正しい。
- 回帰テストの初回で「通常月返済額90,000円台」が一致しなかったのは、共有ドキュメントに残っていたボーナス元金 ¥15,000,000 が通常返済元金を圧縮していたためで、計算ロジックの不具合ではない。ボーナス元金を0に戻すと理論値域（¥90,331）が表示され、計算エンジンの正常動作を確認。
