## テスト結果詳細（Sprint 3 受け入れ基準）

### 合格した項目

1. **3物件比較（月々支払合計の横並び・最安バッジ）**: PASS
   - 物件A（30M/諸経費2M/1%）, 物件B（45M/3M/0%）, 物件C（20M/1M/2%）の3件を登録 →「比較」タブで月々が横並び表示。
   - 月々: 物件A `¥90,331` / 物件B `¥114,286` / 物件C `¥69,565`。
   - 最小値の物件C に「最安 ✓」バッジが緑（`#138A60` 値 + `#1EB980` バッジ）で表示。最安セルのみにバッジが付くことを確認。

2. **空欄の '—' 表示（他物件は正常表示）**: PASS
   - 物件価格を空欄にした物件 → 物件価格・総借入額・月々・総返済額の各セルが `—`。
   - 同表内の値が揃った物件は物件価格 `¥30,000,000`・管理費 `¥12,000`・修繕積立費 `¥8,000` 等を正常表示。空欄物件のセルのみ `—` となり他列レイアウトは崩れない。
   - test1 では3物件すべてに有効値を入れた状態で全セルが実数表示されることも確認済み。

3. **375px レスポンシブ（横はみ出しなし・比較表は横スクロール）**: PASS
   - 375px 幅で `documentElement.scrollWidth = 375 = clientWidth`、`body.scrollWidth = 375`。横方向のはみ出しゼロ。
   - 管理費/修繕積立費 2カラム行の右端 = 183px < ビューポート 375px（`min-w-0` 付与による Sprint 1/2 のバグ#1 解消を確認）。
   - 比較表ラッパーの `overflow-x: auto` を computed style で確認（列が増えても横スクロールで全列閲覧可能）。

4. **PC 1280px レイアウト**: PASS
   - ダーク左サイドバー（物件一覧 + 「物件を追加」）常時表示、入力タブは左フォーム + 右結果の2カラム、比較タブは全幅テーブルで可読。
   - スクリーンショットで配置・余白・選択状態（物件C 緑ハイライト）を確認。

5. **金利0%と金利ありの同時比較**: PASS
   - 物件B（金利0%）`¥114,286`（48,000,000 ÷ 420）と、物件A（1%）`¥90,331`・物件C（2%）`¥69,565` が同一比較表に同時かつ正しく表示。ゼロ除算・NaN なし。

6. **タブ切替の整合**: PASS
   - 入力タブで物件価格を `99,999,999` に変更 → 比較タブに切り替えると同物件の物件価格セルが即 `¥99,999,999` に反映。データの不一致なし。
