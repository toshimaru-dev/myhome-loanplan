**S6-2: 物件URL の入力・保存・リンク**

**S6-3: 新築/中古トグル**

## 改善提案（任意・スコープ外）

- **本番チャンクサイズ**: ビルド時 `index.js` が 734KB（gzip 201KB）。Firebase SDK の取り込みによるもので動作影響はないが、将来的に動的 import によるコード分割の余地あり（Sprint 4 から継続の既知事項、本スプリント対象外）。
- **共有ドキュメント方式の副作用**: 全利用者が `shared/main` を共有するため、評価テスト中に追加した物件が共有データに残る（Sprint 5 で意図的に採用したホワイトボード型の仕様どおり）。テスト後は物件数を整理済み。
- **比較表のメモ非掲載**: メモ行は比較表に含まれない（spec.md の比較表要件は種別/徒歩/URL の3行のみ。仕様準拠）。将来 UX として検討余地あり。

## デザイン仕様適合の所見

- セクション見出し（`font-extrabold 13px text-primary #1EB980`）、トグル選択色（`bg-primary` / 非選択 `bg-toggle #EEF2EE`）、入力フィールド（`bg-bg` / `border-border` / `rounded-field 13px`）はいずれも design.md トークンに準拠。
- design.md にメタUI（URL/トグル/textarea）の個別規定はなく、既存トークンで一貫構成。
- 既存カード「🏠 物件情報」→「🏠 物件価格・諸経費」への改称は progress.md に乖離理由（メタ専用「物件情報」セクション新設との名称衝突回避）を明記済みで、妥当な判断と評価。emoji・カードスタイル・トークンは維持されている。
